Overview

Основные LeanStock workflow разложены по отдельным страницам без лишних demo-блоков.

Production-style demo for LeanStock backend

Frontend consumes the real backend API and demonstrates only the required LeanStock workflows: auth, RBAC, warehouse inventory, reservations, transfers, purchase orders, reports, workers and platform controls.

1. Auth, verification and roles

Без роли пользователь может только register, verify email, login/logout и password reset. Frontend держит только одну активную login-сессию: новый login очищает прошлую роль.

Password reset

Проверка обязательного workflow из ТЗ: reset link/token отправляется на реальный email через worker. После reset backend инвалидирует старые токены, а frontend сразу разлогинивает текущую сессию.

2. Warehouse, catalog, product and inventory

Основной LeanStock flow: склады, поставщик, категория, продукт, партия товара, recount adjustment.

Uses active tenant role

Warehouses

OWNER/MANAGER создают и обновляют склады. Write actions enqueue real email to OWNER/MANAGER. STAFF/AUDITOR должны получить 403.

Supplier & category

Нужно для product catalog и purchase order workflow. Supplier/category are CRUD actions; required emails are low-stock, PO confirmation and transfer receipt.

Product catalog

Полный multi-tenant product catalog с CRUD: create, read, update и archive/delete. Mutation buttons enqueue real email notifications.

Inventory batch, adjustment & issue approval

Приём партии товара, direct manager adjustment, and STAFF damaged/missing report for manager approval. Write actions enqueue real email; low-stock also triggers required manager alert.

Quick data loading

Эти кнопки помогают подтянуть IDs из базы, если страница перезагрузилась.

Если action требует productId/warehouseId/supplierId, сначала создай или загрузи соответствующие данные. Email по ТЗ приходит не на каждое действие, а на нужные business events: low-stock, transfer receipt, purchase order confirmation.

3. Reservation, sale, transfer and purchase order

Три business workflows для финальной защиты: checkout reservation, inter-location transfer, supplier purchase order.

Reservation checkout

Создаёт reservation с idempotency key и Redis lock на inventory.

Transfer state machine

Blueprint flow: REQUESTED → APPROVED → IN_TRANSIT → RECEIVED. Stock moves only on Receive.

Purchase order

Supplier workflow: create, confirm, cancel, list.

Sales

Sales list and manual real email notification allowed only for OWNER/MANAGER. Manual email is queued by worker.

4. Reports, audit and background jobs

Отчёты, worker queue visibility и dead-stock background processing.

Reports

Доступны OWNER, MANAGER, AUDITOR. AUDITOR читает, но не меняет данные.

Dead stock worker & policy

Configurable tenant policy + immediate decay or Redis/BullMQ queued job.

Audit logs

Журнал действий: кто и что сделал в tenant.
